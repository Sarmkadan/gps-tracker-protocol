# ResponseMessage

A data transfer object used to encapsulate responses received from GPS tracking devices, including command acknowledgments, telemetry data, and error states. It standardizes the format of device-originated messages for parsing, validation, and downstream processing.

## API

### `public string Id`
Unique identifier for the response message, typically generated by the originating device. Used to correlate responses with requests or track message flow in device logs.

### `public string DeviceId`
Identifier of the GPS device that generated the response. Required for routing and device-specific processing pipelines.

### `public string? CommandId`
Optional identifier of the command that triggered this response. Present when the response is a direct reply to a prior command; otherwise `null`.

### `public MessageType Type`
Enumerated value indicating the semantic category of the response (e.g., `Telemetry`, `Ack`, `Error`). Determines how downstream systems interpret `Content` and `ParsedData`.

### `public bool IsSuccess`
Indicates whether the operation or command succeeded. When `false`, `ErrorCode` and `ErrorMessage` should be inspected for diagnostic details.

### `public string Content`
Raw payload received from the device. May be JSON, plaintext, or binary depending on device firmware and protocol version. Must be parsed via `Parse()` to populate `ParsedData`.

### `public Dictionary<string, object> ParsedData`
Structured representation of `Content` after successful parsing. Populated by calling `Parse()`. Contains device-specific key-value pairs (e.g., latitude, speed, battery level). Empty if parsing fails or not attempted.

### `public DateTime ReceivedAt`
Timestamp indicating when the response was received by the host system. Used for ordering, latency calculation, and time-series analysis.

### `public int ErrorCode`
Numeric error identifier when `IsSuccess` is `false`. Zero indicates success; non-zero values are protocol- or device-specific.

### `public string? ErrorMessage`
Human-readable error description when `IsSuccess` is `false`. May be `null` if the device did not provide a message.

### `public bool IsValid`
Indicates whether the response conforms to expected structural and semantic rules (e.g., required fields present, `Type` is recognized). Set during validation logic; does not imply command success.

### `public void Parse()`
Parses `Content` into `ParsedData` using protocol-specific rules inferred from `Type`. Throws `FormatException` if `Content` is malformed or incompatible with `Type`. No-op if already parsed or `Content` is empty.

### `public override string ToString()`
Returns a human-readable summary of the response, including `Id`, `DeviceId`, `Type`, `IsSuccess`, and `ErrorCode` (if applicable). Useful for logging and debugging.
